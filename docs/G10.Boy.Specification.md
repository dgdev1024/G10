# Specification: G10.Boy

Below is a specification for the "G10.Boy", a virtual hardware implementation
powered by the G10 Virtual CPU, and based tightly on the Nintendo Game Boy Color
hardware.

## 2. Overview

The **G10.Boy** is a virtual hardware implementation based upon the Nintendo 
Game Boy Color, powered by the 32-bit G10 Virtual CPU instead of the original 
8-bit Sharp LR35902. It retains, as much as possible, compatibility with Game 
Boy Color software conventions while leveraging the enhanced capabilities of the 
G10 CPU to provide improved performance and expanded address space.

### 2.1 Key Characteristics

- **CPU**: 32-bit G10 Virtual CPU; 16 32-bit registers; 32-bit address space.
- **Display**: 160 x 144 pixel LCD display with support for both monochrome
    (DMG) and color (CGB) rendering modes.
- **Memory**: 32-bit flat address space (Up to 4 GB addressable).
- **Audio**: 4 channel audio processing unit, with pulse, wave and noise
    synthesis.
- **Graphics**: Tile-based background/window system with memory for up to 40
    objects, 10 per scanline.
- **Timekeeping**: Cycle-accurate internal timer with system counter and divider.
- **I/O Ports**: 256 bytes reserved for memory-mapped I/O registers in the upper
    memory region.

### 2.2 Hardware Specifications

- **Processing Power**:
    - CPU Clock: 4.194304 MHz (Normal Speed); 8.388608 MHz (High Speed)
    - Component Clock: 2.097152 MHz (Normal Speed); 4.194304 MHz (High Speed)
        - 1 Component Clock ("Dot") per 2 CPU Clocks
        - Component Clock occurs every other CPU Clock
        - *Rationale*: The G10 CPU consumes roughly twice the number of M-cycles
            as the Game Boy's CPU (due to all opcodes being two bytes instead of
            one, up to four-byte immediate fetches, etc.). As such, the G10.Boy's
            components should be clocked at *half* the CPU's clock speed in order
            to maintain a cycle accuracy approximately equal to the original
            Game Boy hardware.
    - Data Bus: 32-bit virtualized
    - Address Bus: 32-bit (4 GB addressable)
    - Registers: 16x 32-bit registers
        - Accessible in full as 32-bit values (`D0` to `D15`)
        - Accessible in part as 16-bit values (`W0` to `W15` - lower 16 bits 
            of `Dn`)
        - Accessible in part as 8-bit values (`H0` to `H15` - upper 8 bits of
            `Wn`; `L0` to `L15` - lower 8 bits of `Wn`)
- **Display**:
    - Resolution: 160 x 144 pixels
    - Color Depth: 15-bit (32,768 colors) in CGB palette mode; 4-bit monochrome
        in DMG palette mode.
    - Refresh Rate: ~59.7 Hz (~16.74 ms per frame)
        - The exact rate here may differ due to a somewhat different
            cycle timing on account of the G10 CPU and the G10.Boy's component
            clock.
    - PPU Component Clock: 2.097152 MHz (Both Speeds)
        - 1 PPU Clock per 2 CPU Clocks in Normal Speed
        - 1 PPU Clock per 4 CPU Clocks in High Speed
    - Scanlines per Frame: 154 (144 visible + 10 vertical blank)
- **Memory Configuration**:
    - Program ROM: Up to 2 GB, flat-mapped
    - Video RAM (VRAM): 8 KB (2 banks; switchable)
    - Object Attribute Memory (OAM): 160 Bytes
    - External ("Save") RAM (SRAM): Up to 256 MB (defined in program metadata)
    - Work RAM (WRAM): Up to 256 MB (defined in program metadata)
    - I/O Registers: 256 Bytes Reserved
    - Quick RAM (QRAM) / High RAM (HRAM): 64 KB

## 3. Differences and Modifications

To accomodate with the G10 Virtual CPU, among other quality-of-life changes, the
G10.Boy implements a number of differences, modifications and deviations from
the Game Boy (Color) hardware, including, but not limited to, the following:

- **32-Bit Processing**: Instead of the Sharp LR35902, the G10.Boy is powered by
    the 32-bit G10 Virtual CPU, with 16 general-purpose registers which can be
    accessed in full or in part.
- **Expanded Interrupt Lines**: The G10 CPU provides 32 interrupt request lines,
    including one NMI exception handler, with two 32-bit interrupt control
    registers (`IE` and `IRQ`), accessible via eight 8-bit hardware registers
    (`IE0` through `IE3` and `IRQ0` through `IRQ3`).
- **Flat Address Space**: The 32-bit memory map eliminates the need for banking
    and bank switching for ROM and WRAM. Memory is accessed linearly from
    `$00000000` to `$FFFFFFFF`.
- **VRAM Banking**: VRAM remains banked to maintain conformity with the Game Boy's
    PPU and rendering specifications.
- **Larger I/O Region**: Expanded from 128 bytes to 256 bytes, providing space 
    for both legacy and G10.Boy-specific registers.
- **Register Consolidation**: `PPUX` consolidates `VBK` (VRAM Bank Select) and 
    `OPRI` (Object Priority Mode), and introduces a Palette Mode bit. 
    `BANK` and `SVBK/WBK` are removed entirely.

## 4. Memory

The G10.Boy implements a 32-bit memory layout built on top of the G10 CPU's
broadly-defined memory layout. This layout is defined in the table below:

| Start       | End         | Description                                       |
|-------------|-------------|---------------------------------------------------|
| `$00000000` | `$7FFFFFFF` | Program ROM (Up to 2 GB)                          |
| `$80000000` | `$9FFFFFFF` | Special-Purpose RAM Region                        |
| `$80008000` | `$80009FFF` | - Video RAM (8 KB, switchable bank 0/1)           |
| `$8000FE00` | `$8000FE9F` | - Object Attribute Memory (160 Bytes)             |
| `$8000FF30` | `$8000FF3F` | - Wave Pattern Memory (16 Bytes)                  |
| `$A0000000` | `$BFFFFFFF` | General-Purpose RAM Region                        |
| `$A0000000` | `$AFFFFFFF` | - Work RAM (Up to 256 MB)                         |
| `$B0000000` | `$BFFFFFFF` | - Save RAM (Up to 256 MB)                         |
| `$C0000000` | `$FFFEFFFF` | Expansion Memory Region                           |
| `$FFFF0000` | `$FFFFFFFF` | Quick RAM                                         |
| `$FFFFFF00` | `$FFFFFFFF` | I/O Registers                                     |

## 5. I/O Ports

The G10.Boy's virtual hardware implementation maps certain hardware registers
to addresses in the **I/O Registers** region of its memory layout. These port
registers include most (but not all) of the I/O registers available on the
original Game Boy and Game Boy Color hardware, as well as additional registers
to accomodiate the G10 CPU and its 32-bit address space, and some custom
hardware exclusive to the G10.Boy.

### 5.1 I/O Ranges

The G10.Boy uses the I/O ranges laid out in the table below. Addresses therein
are relative to the I/O Registers region starting at `$FFFFFF00`.

| Start | End   | Description                                                   |
|-------|-------|---------------------------------------------------------------|
| `$00` | `$0F` | G10 CPU Hardware Registers                                    |
| `$10` | `$1F` | Timekeeping Hardware Registers                                |
| `$20` | `$2F` | Communications Hardware Registers                             |
| `$30` | `$5F` | APU Hardware Registers                                        |
| `$60` | `$6F` | PPU Hardware Registers                                        |
| `$70` | `$8F` | Direct Memory Access (DMA) Registers                          |
| `$90` | `$9F` | Input Hardware Registers                                      |
| `$A0` | `$FF` | User-Defined Hardware Registers                               |

### 5.4 New Registers

The following I/O port registers are exclusive to the G10.Boy virtual hardware,
and are not present in the official Game Boy Color hardware:

- `RTC*`: Real-Time Clock Registers
    - These registers are provided by the G10.Boy's real-time clock component.
    - This component is based on the Game Boy MBC3's RTC hardware.
    - `RTCS` holds latched seconds.
    - `RTCM` holds latched minutes.
    - `RTCH` holds latched hours.
    - `RTCDL` and `RTCDH` hold the low and high bytes, respectively, of the
        16-bit day counter.
    - `RTCC` holds the control register for the RTC.
        - `RTCC.5` dictates how the RTC should latch values.
            - Set: Latch values according to the host system's time.
            - Clear: Latch values according to the RTC's time registers.
        - `RTCC.6` dictates whether the RTC is halted.
            - Set: Halt the RTC (do not update registers automatically).
            - Clear: Enable the RTC (update registers automatically).
    - `RTCL`: Write-only. Any write to this register latches the current RTC values.
- `PPUX`: PPU Auxillary Settings
    - This register contains additional settings for the PPU.
    - Bit `0`: VRAM Bank Select (the CGB's `VBK` register)
        - Controls which VRAM bank is currently active.
        - Set: Use VRAM Bank 1.
        - Clear: Use VRAM Bank 0.
    - Bit `1`: Object Priority Mode (the CGB's `OPRI` register)
        - Determines the order in which objects on the current scanline should 
            be rendered.
        - Set: Order based on object X position, then on OAM index (DMG-style priority)
        - Clear: Order based only on OAM index (CGB-style priority)
    - Bit `2`: Palette Mode
        - Determines what palette system the PPU should use to render graphics.
        - Set: Use color palette RAM (CGB-style rendering)
        - Clear: Use monochrome palettes (DMG-style rendering)

### 5.5 Removed Registers

The following I/O port registers which are present in the official Game Boy
Color hardware, are not present in the G10.Boy virtual hardware:

- `KEY0` / `SYS`: CPU Mode Select
    - This register was used by the Game Boy Color to determine whether or not
        DMG compatibility mode should be used for a DMG-only cartridge.
    - Removed because the G10.Boy virtual hardware is based on the Game Boy
        Color hardware, and not the original DMG Game Boy.
- `VBK`: VRAM Bank Select
    - This register was used by the Game Boy Color to select the active VRAM bank.
    - Removed as a standalone register. In G10.Boy, the lone meaningful bit of
        this register is part of the `PPUX` PPU Auxillary Settings register.
- `BANK`: Boot ROM Mapping Control
    - This register is written to by the Game Boy's boot ROM in order to un-map
        it from the address space.
    - Removed because the G10.Boy virtual hardware does not currently support a
        boot ROM. This is subject to change, however, and this register may be
        added back in a future version of G10.Boy.
- `OPRI`: Object Priority Mode
    - This register was used by the Game Boy Color's PPU to determine the order
        in which objects on the current scanline should be rendered.
    - Removed as a standalone register. In G10.Boy, the lone meaningful bit of
        this register is part of the `PPUX` PPU Auxillary Settings register.
- `SVBK` / `WBK`: WRAM Bank Select
    - This register was used by the Game Boy Color to select the active WRAM bank.
    - Removed because the G10.Boy virtual hardware's work RAM is flatly-mapped,
        and does not require bank selection/switching.

## 6. Program

### 6.1 Program Metadata

G10 CPU executables for use by the G10.Boy system are expected to contain the
following information in their metadata section. Addresses shown are absolute.

| Address | Type        | Description                                                       |
|---------|-------------|-------------------------------------------------------------------|
| `$0000` | String      | Magic Number (ASCII "G10.BOY", Null-Terminated)                   |
| `$0008` | UInt32      | Requested WRAM Size (In Bytes, Max `0x10000000`)                  |
| `$000C` | UInt32      | Requested SRAM Size (In Bytes, Max `0x10000000`)                  |
| `$0010` | String      | Program Title (ASCII, Null-Terminated, Max 32 Bytes)              |
| `$0030` | String      | Program Author (ASCII, Null-Terminated, Max 32 Bytes)             |
| `$0050` | String      | Program Description (ASCII, Null-Terminated, Max 64 Bytes)        |
